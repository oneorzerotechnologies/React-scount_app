Design notes

Decisions baked into these mockups

  • Brand consistency. Same `moss` palette, Inter typeface, gradient mark, and light/dark switch as the marketing site — feels like one product.
  • Tap targets ≥ 44pt. Every button hits Apple HIG and Material Design minimums. List rows are 56pt tall.
  • Splash, not landing. Screen 01 has no CTAs. Held 5 seconds while we validate the stored token, then auto-routes — login if signed out, workspace or dashboard if signed in.
  • "Skip when there's only one." Workspace screen is gated by count. Single-workspace users go straight from login to dashboard. Multi-workspace users always tap explicitly — no auto-advance.
  • Cash on hand on top. The number people open the app for. Everything else is supporting context.
  • Three quick actions only. Invoice / Expense / Customer — the three verbs that justify a mobile app. More buttons = more friction.
  • Bottom tab bar: Home · Quotation · Invoice · Contacts · More. "Contacts" is the umbrella for everyone you transact with. Inside the tab, a segmented control flips between Clients and Suppliers. Expenses, reports, fixed assets, reconciliation live behind More.
  • Biometric only after first login. Day-1 users still need to type the password once, so support can verify them; biometric kicks in from session 2.
