Notes

Two contexts, one tab bar. 08A's tab bar shows More active because the user is sitting on that tab. 08B's tab bar shows Home active because Notifications is a stack screen pushed onto Home from the bell icon — popping it returns to the dashboard. Both keep the bar visible per the design rule.

Categories & mute toggles. Five categories, each individually mutable from the Notifications sub-screen (linked from More): Quote accepted, Quote viewed, Invoice paid, Invoice overdue, Weekly digest. The summary "5 of 5 on" line on More tells the user at a glance whether anything is muted.

"Get the rest on scount.my". The deep-link card opens the workspace web app in an in-app browser session. Same Sanctum token via short-lived web SSO — the user lands signed-in. Used for full reports, bank reconciliation, multi-line journals, year-end closing — anything intentionally kept off mobile.

Sign out is destructive-styled. Red border + red text matches the platform convention for irreversible-ish actions. Tap surfaces a confirm sheet ("Sign out of Eva@acme.co?") so a fat-fingered tap doesn't dump the user back to the login screen.

No payment / reminder shortcuts here either. The More menu intentionally does not surface payment recording or reminder sending — those are web-only on v1, consistent with the invoice detail.
