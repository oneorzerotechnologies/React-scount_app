Notes

Same schema, different lens. The Contacts API is one endpoint with a type filter — Clients and Suppliers share fields. Type is set at create time and immutable; an org can be both, but it'll be two records.

Pastel avatars are deterministic. First letter colours are derived from a hash of the contact ID — same contact, same colour, every device.

Suppliers in v1 are read-only re: bills. You can create and view suppliers in v1, but bill management lives in Phase 2. The detail screen for suppliers in v1 shows an "Coming soon" empty state where the bills list would be.
