Notes

CRUD on mobile. View = detail screen (05B). Add = create screen (05C). Edit = the same create form, opened pre-filled via the pencil icon in the detail header — header swaps to "Edit · QT-036", Save calls PATCH /v1/quotations/{id}. Delete sits in the three-dots overflow, behind a confirmation sheet, and is blocked once the quote has been converted to an invoice.

Additional information. Four fields ride alongside line items: Delivery days (e.g. "30 days from acceptance"), Terms & conditions, Remarks (visible on the PDF the customer receives), and Internal remarks (PDF-hidden, marked with the amber PDF-hidden badge). All four round-trip through both the create form and the detail view; none are lost on convert-to-invoice.

Convert to invoice = one tap. On the detail screen for an ACCEPTED quote, the primary CTA fires POST /v1/quotations/{id}/convert-to-invoice and routes the user to the new invoice's detail. The quote becomes read-only and shows a "Converted to INV-00510 →" card. Terms, remarks, and internal remarks are copied across; the user can edit them on the new invoice.

Status pill colours. OPEN slate, ACCEPTED moss-green, DECLINED red, EXPIRED slate (dimmed), CONVERTED moss-green with a chain icon.

Share is always one tap. The Share button in the action bar opens the OS share sheet with share_url from the detail response (a signed link to scount.my/q/<token>). Internal remarks are stripped from the rendered PDF, so the customer never sees them.

Tax follows the web. Whether each line item shows a tax pill, whether the totals card has a Tax row, and whether the create form offers a tax-code picker — all of it reads from workspace.settings.tax_enabled on scount.my. If the workspace runs tax-disabled, the app suppresses every tax affordance and tax_minor stays 0. Mobile cannot override the setting; the user toggles it on web and the app picks it up on next refresh.
